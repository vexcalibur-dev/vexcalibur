# Security policy

## Report a vulnerability

Use [GitHub private vulnerability reporting](https://github.com/vexcalibur-dev/vexcalibur/security/advisories/new) for a Vexcalibur vulnerability. The organization-wide policy is at [vexcalibur-dev/.github](https://github.com/vexcalibur-dev/.github/security/policy).

Do not put vulnerability details, exploits, secrets, tokens, private package names, logs, stack traces, screenshots, or reproduction steps in a public issue. If private vulnerability reporting is unavailable, use the [private channel request form](https://github.com/vexcalibur-dev/vexcalibur/issues/new?template=private_disclosure_request.yml) without sensitive details.

## Supported versions

Security fixes target `main` and the latest published release. Older pre-1.0 releases do not receive separate maintenance updates.

| Version | Security fixes |
| --- | --- |
| Latest release | Yes |
| `main` | Yes |
| Older releases | No |
